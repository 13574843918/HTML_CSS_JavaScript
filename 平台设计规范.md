# HTML+CSS 网页设计实训平台 - 设计规范

> 本文档是平台开发与维护的设计规范，适用于新建页面、修改现有页面或扩展功能时参考。
> 最后更新：2026-04-09

---

## 一、项目结构规范

```
web-training-platform/
├── index.html          # 主页（任务入口页面）
├── task1.html          # 任务一：CSS基础与选择器
├── task2.html          # 任务二：字体与文本样式
├── task3.html          # 任务三：选择器与层叠性
├── styles.css          # 全局共享样式（变量、布局、通用组件）
├── script.js           # 全局共享脚本（交互功能、进度追踪）
└── .workbuddy/         # AI工作记忆（勿动）
```

- 所有 HTML 文件均引用同一份 `styles.css` 和 `script.js`
- 不允许在 task 页面中直接修改 `styles.css` / `script.js` 中已实现的功能，如需扩展应先在规范中新增条目
- 图片资源（未来扩展）统一放在 `images/` 目录下

## 四、JavaScript 规范（script.js）

### 4.1 初始化函数调用顺序

DOMContentLoaded 中按以下顺序调用各模块：

```javascript
// 1. 进度追踪
ProgressTracker.init();

// 2. 移动端菜单
initMobileMenu();

// 3. 导航高亮
initNavigation();

// 4. 案例折叠
initExampleCards();

// 5. 代码高亮（可选）
initCodeHighlight();

// 6. 复制按钮
initCopyButtons();

// 7. 实时预览
initLivePreviews();

// 8. 案例内HBuilderX技巧
initHBuilderTips();

// 9. 知识点外链（必须调用，勿删除）
initKnowledgeLinks();
```

> **新增功能时**：必须在初始化函数中添加调用，并确保功能模块的函数已定义。

### 4.2 实时预览功能

```javascript
// 监听所有 .code-editor
document.querySelectorAll('.code-editor').forEach(editor => {
    editor.addEventListener('input', function() {
        const previewId = this.id.replace('code-', 'preview-');
        const preview = document.getElementById(previewId);
        if (preview) {
            preview.srcdoc = this.value;
        }
    });
    // 初始化时执行一次
    editor.dispatchEvent(new Event('input'));
});
```

### 4.3 导航高亮

根据当前 URL 自动高亮对应的导航项：

```javascript
function initNavigation() {
    const currentPage = window.location.pathname.split('/').pop() || 'index.html';
    document.querySelectorAll('.nav-item').forEach(item => {
        if (item.getAttribute('href') === currentPage) {
            item.classList.add('active');
        }
    });
}
```

### 4.4 知识点外链（MDN文档）

`initKnowledgeLinks()` 自动为带 `knowledge-link` 类的元素添加 MDN 外链：

```javascript
function initKnowledgeLinks() {
    document.querySelectorAll('.knowledge-link').forEach(link => {
        const keyword = link.getAttribute('data-keyword');
        if (keyword) {
            link.href = `https://developer.mozilla.org/zh-CN/docs/Web/CSS/${keyword}`;
            link.target = '_blank';
        }
    });
}
```

**使用方式**：在知识点 `<code>` 标签上添加 `class="knowledge-link"` 和 `data-keyword="属性名"`：

```html
<code class="knowledge-link" data-keyword="font-size">font-size</code>
```

支持的 MDN 关键词包括：`style`, `font-size`, `color`, `font-family`, `text-align`, `line-height`, `font-weight`, `selector`, `class-selector`, `id-selector`, `descendant-selector`, `border`, `background-color`, `padding`, `margin`, `display`, `box-sizing`, `border-radius`, `width`, `height`, `z-index`, `position`, `overflow`, `flex`, `grid`, `transform`, `transition`, `animation`, `media-queries` 等。

## 七、颜色使用规则

- **主题色**：橙色 `#f97316` 作为平台统一强调色（主页、按钮、高亮）
- **任务色**：每个 task 页面有独立主题色（task1蓝紫、task2橙、task3绿），仅在案例相关元素上使用
- **背景**：深色主题，`#0a0e17` 为最深层，`#141c2b` 为卡片层
- **文字**：`#f1f5f9` 主文字，`#94a3b8` 次要文字，`#64748b` 辅助文字
- **边框**：`#2d3748` 常规边框，`#374151` 悬停边框

## 八、修改现有页面的注意事项

1. **只改内容不改样式**：除非用户明确要求，不改变现有 CSS 类和结构
2. **保持一致性**：新增案例时，复制现有 `.example-card` 结构并修改内容
3. **外链安全**：知识点外链使用 MDN 中文文档，格式固定，勿链接到其他站点
4. **初始化检查**：在 `script.js` 新增全局函数后，必须在 DOMContentLoaded 中调用
5. **移动端测试**：修改布局后用 768px 断点验证移动端效果

## 九、快速参考：新建一个 task 页面

1. 复制 `task2.html` 作为模板
2. 修改 `<title>` 和 `<style>` 中的主题色
3. 在 `index.html` 的任务网格中添加入口卡片
4. 添加页面级 HBuilderX 技巧（选择当前任务未使用的主题）
5. 填充案例内容，确保每个案例有描述、代码+预览、知识点、练习
6. 验证实时预览功能正常
